# FoodConnect Application Flow

This document details the complete operational lifecycle of the FoodConnect platform, from user onboarding to the final delivery and impact reporting.

## Phase 1: User Onboarding and Identity Verification

1.  **Registration/Login (Backend-Based)**: Both Volunteers and Hotel Partners navigate to the web portal. The system relies on a PostgreSQL database for authentication.
2.  **Role Selection & Data Entry**: Users select their role ("hotel" or "volunteer") and provide necessary details (e.g., license numbers for hotels, vehicle types for volunteers).
3.  **Database Mapping & Routing**: The custom backend validates credentials and creates/checks the `users` table along with the associated `hotel` or `volunteer` tables. Upon login success, the system dynamically routes the user to either the Hotel App Layout or Volunteer App Layout based on their stored role.

## Phase 2: Donation Generation (Hotel Flow)

1.  **Post Creation**: The Hotel Partner opens the "Donate" page on their dashboard.
2.  **Data Input**: They enter surplus food specifics:
    *   Total weight (in kg/lbs)
    *   Categorical tags (e.g., biryani, rice, dal)
    *   Pickup expiration window (urgency indicator)
3.  **Database Commit**: The frontend submits this data to the backend API.
4.  **Transaction**: A new record is inserted into the `donations` table, and associated tags are updated in the `donation_tags` table.
5.  **Status**: The donation status is initialized as `PENDING`.

## Phase 3: Volunteer Discovery (Volunteer Flow)

1.  **Live Mapping Interface**: Volunteers access a Leaflet-powered map displaying currently active donations. The map uses clustering algorithms to handle high densities of pins efficiently.
2.  **Routing & Decision Making**:
    *   The volunteer taps a donation pin.
    *   The application pings the OSRM (Open Source Routing Machine) API to calculate the *actual* road distance and Estimated Time of Arrival (ETA) from their current GPS location to the hotel.
3.  **Claim Request**: The Volunteer selects "Request Pickup". The system logs a claim request, marking the donation's state to await hotel approval.

## Phase 4: Hotel Approval and Secure Handoff Setup

1.  **Claim Notification**: The Hotel dashboard updates to show the specific volunteer request for the pending donation.
2.  **Acceptance & Verification Setup**:
    *   The Hotel reviews the volunteer details (vehicle type, gamification score) and clicks "Accept".
    *   The donation status shifts to an active delivery state.
    *   **Crucial Step**: The system generates a cryptographic OTP (One-Time Password) representing a secure handoff mechanism.
3.  **Real-Time Tracking Initiation**: The volunteer's application begins broadcasting their location to the backend`/location/update` endpoint (e.g., every 10 seconds). The hotel can view this progress live.
4.  **Volunteer Notification**: The volunteer receives an FCM (Firebase Cloud Messaging) push notification confirming the pickup and their UI enters Navigation Mode.

## Phase 5: Handoff and Completion

1.  **Physical Arrival & Navigation**: The Volunteer follows turn-by-turn navigation provided by the app to reach the Hotel.
2.  **Handshake Verification**: Upon arrival, the Volunteer presents their app screen, which securely displays the 4-digit Authorization OTP.
3.  **Final Authorization**: The Hotel representative inputs this specific OTP into their portal. The backend validates this code.
4.  **State Resolution**: If the OTP matches, the database transactions finalize:
    *   The specific `claims_record` is marked as `DELIVERED`.
    *   The volunteer's gamification points (`score`) increment in the `volunteer` table.
    *   Notifications of successful delivery are dispatched to both parties.

## Phase 6: Impact and Reporting

1.  **History Generation**: The completed transaction moves to the "Donation History" tab for audit purposes.
2.  **ESG Reporting**: Hotels can utilize a 1-click export feature (CSV/Google Sheets) summing total weight donated, recipient info, and timestamps for tax/Corporate Social Responsibility (CSR) compliance.
3.  **AI Video Generation**: Integration with the Google Gemini API allows hotels to request dynamic "Impact Summary Reports" or marketing video scripts generated by analyzing donation history and uploaded food imagery.
