# FoodConnect Development and Deployment Guide

## Development Environment Setup

### Prerequisites
- Node.js v18+ (LTS recommended)
- npm v9+ or yarn v1.22+
- Python 3.8+ (for backend API)
- PostgreSQL 13+ (local or Docker)
- Git for version control

### 1. Clone and Install Dependencies
```bash
git clone https://github.com/your-repo/foodconnect.git
cd foodconnect
npm install
```

### 2. Environment Configuration
Create `.env.local` in project root:
```env
# Supabase Configuration
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key

# Firebase Configuration
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.firebasestorage.app
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id

# OpenRouteService API Key
VITE_ORS_API_KEY=your_openrouteservice_key

# Backend Configuration (if using FastAPI)
DATABASE_URL=postgresql://user:password@localhost:5432/foodconnect
JWT_SECRET=your_secure_jwt_secret
```

### 3. Database Setup
```bash
# Using Docker
docker run --name foodconnect-db -e POSTGRES_PASSWORD=postgres -p 5432:5432 -d postgres:13

# Or install PostgreSQL locally
# Then run the schema
psql -U postgres -d foodconnect -f database/queries/Sql_Queries.sql
```

### 4. Backend Setup (Optional)
```bash
cd backend
pip install -r requirements.txt
# Set environment variables in backend/.env
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 5. Development Commands
```bash
# Start development server
npm run dev

# Start backend API
npm run api

# Type checking
npm run lint

# Build for production
npm run build
```

## Database Schema Management

### Schema Overview
The application uses a normalized PostgreSQL schema with 7 core tables:
- `users` - Central user registry
- `hotel` - Hotel partner profiles
- `volunteer` - Volunteer profiles
- `donations` - Food donation records
- `donation_tags` - Multi-valued food categories
- `claims_record` - Volunteer claims on donations
- `verification` - OTP verification records

### Database Operations
- **Migrations**: Version-controlled SQL scripts
- **Backup**: Automated via Supabase
- **Testing**: `scripts/verify_db.py` for connection validation
- **Performance**: Strategic indexing for queries

## Troubleshooting

### Common Issues

#### Database Connection Issues
```bash
# Test connection
python scripts/verify_db.py

# Check PostgreSQL logs
docker logs foodconnect-db

# Verify environment variables
echo $DATABASE_URL
```

#### Real-time Sync Issues
- Check Supabase dashboard for connection status
- Verify WebSocket connections in browser dev tools
- Test with localStorage fallback

#### Authentication Issues
- Verify Firebase configuration
- Check phone number formatting
- Validate JWT token expiration

#### Map Loading Issues
- Verify OpenRouteService API key
- Check Leaflet tile server connectivity
- Test with fallback map providers
